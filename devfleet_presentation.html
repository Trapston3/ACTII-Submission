Efficiency

The 1-Token Verification Gate

How the 1-Token Gate works

  • Borderline cases: If local sentiment confidence is borderline (0.90–0.95), the engine does not issue a full completion request.
  • 1-Token Constrained Call: It issues a prompt to a cheap model (e.g. glm-5p1) formatted to output only `YES` or `NO`.
  • Server Forced Parameter: Configures max_tokens: 1 and logit biases on the API call.
  • Verification Result: On `YES`, local sentiment solution is written. On `NO`, the query escalates to the Tier 3 model.

Token Cost Comparison

Deployment

VM Hardening & Packaging

# Multi-stage Scratch Dockerfile
FROM golang:1.21-alpine AS builder
WORKDIR /app
COPY . .
RUN CGO_ENABLED=0 GOOS=linux GOARCH=amd64 go build \
    -ldflags="-s -w" -o agent ./cmd/agent

FROM scratch
COPY --from=builder /etc/ssl/certs/ca-certificates.crt /etc/ssl/certs/
COPY --from=builder /app/agent /agent
ENTRYPOINT ["/agent"]

Key Hardening Points

  • Under 12MB Image: The scratch build drops all OS bloat, preventing VM loading bottlenecks.
  • Local Root Certs: Statically links SSL CA certificates to prevent TLS errors in sandbox.
  • 5xx Fallback Mesh: Router automatically detects remote API timeouts or server errors and retries on alternate model channels.
